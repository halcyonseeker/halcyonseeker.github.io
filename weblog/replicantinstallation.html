Android Replicant on a Galaxy Nexus: Installaton

Instructions are specifically for the Galaxy Nexus i9250, they will not work for other devices

For this guide I am assuming that you are using and reasonably familiar with GNU/Linux.

Download and Verify Recovery and OS Images

In order to ensure that the files you download were not corrupted or tampered with, add the release key to you keychain, as per this guide. I'm using Replicant 6.0, so I want the key with ID 5816A24C10757FC4. 

# Import the key
gpg --recv-keys 5816A24C10757FC4

Next, download the appropriate recovery image, the OS image, their checksum file, and signature files from here. For me they are:

  • replicant-6.0-maguro.zip – The OS.
  • replicant-6.0-maguro.zip.asc – The OS signature file.
  • recovery-maguro.img – The recovery.
  • recovery-maguro.img.asc – The recovery signature file.
  • maguro.sha256 – The checksum file for the OS and recovery.

Verify the recovery and OS signature files. If you do not see good signature DO NOT PROCEDE, the files have been tampered with. 

# Recovery
gpg --verify recovery-maguro.img.asc
gpg --verify replicant-6.0-maguro.zip.asc

You next should verify the checksums of the OS and recovery files to ensure that they have not been corrupted in transit. In order to avoid tedious manual comparison between the output of sha256sum and the content of maguro.sha256, I recommend using the script below.

If all went well, you're all set to begin flashing the recovery!

Flash Recovery Image

In order to proceed with flashing the recovery image, you'll need to install heimdall, in Arch GNU/Linux it is available in the AUR.

In order to prepare to flash the recovery, shut down your phone then boot into heimdall mode by holding down the power and volume down buttons at the same time until you see green START. At this point, plug the phone into your computer and run the following command: 

sudo heimdall flash --boot /path/to/recovery-maguro.img --recovery /path/to/recovery-maguro.img

Upon rebooting your phone you should now find yourself in the Replicant Recovery.

Next use the recovery to wipe the system, cache, and data partitions. This can be done selecting Advanced > Wipe system partition and Factory Reset > Wipe data (keep media) respectively. You should now be ready to install the operating system.

Flash OS

This is where I had a bit of trouble. All the instructions on redmine.replicant.us required an SD card, which the Galaxy Nexus 19250 does not have; an installed OS, which was wiped after the previous step; or straight up did not work.

So after a bit of research, one deep dead end, and mild panic that I had ruined yet another perfectly good device by being to hasty, I found the solution here on Replicant's message board.

So instead of retrying the rather poor installation instructions I used the ADB, or Android Debug Bridge to load it. The ADB is this cool system for working with Android phones the way God and Dennis Ritchie intended, from a command line on a proper computer.

First, you need to install the ADB. In Arch it can be found in the android-tools package.

To flash, or sideload, the OS plug it into your computer and select Apply update > Apply from ADB then run the following ADB command: 

adb sideload /path/to/replicant-6.0-maguro.zip

Your phone will switch to a screen with a wiggly astromech looking thing and progress bar. After a few minutes the previous command will output Total xfer: 1.00x and your phone will switch back to the recovery screen. Now you can factory reset it with Factory reset > Full factory reset.

Unplug your phone and hit Reboot system now in the main menu of the recovery.

If all went well, you should be booted into Android Replicant!
