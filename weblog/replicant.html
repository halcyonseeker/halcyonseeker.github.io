Then one day I was cruzing around fsf.org looking for cool stuff, and I happened upon Andorid Replicant, a distribution of Andorid (aka "ROM") built to run with as little proprietary firmware as possible.

Android Replicant

For those not familier with Android, the kernel and a lot of userspace software is open-source, and there is a fairly sizable ecosystem of free and open source software available from the Fdroid repository. The catch is in the hardware. Most hardware requires large proprietary blobs, some of which needs to run on the CPU along side the OS, where others can run on external processors like the SIM card and the baseband moden (which is usually built into the CPU, but whatever). In order to be endorsed by the Free Software Foundation, Replicant needs to depend on as little proprietary blobs as possible. This makes the hardware selection rather limited, with about a dozen supported devices from the early 2010s.

I went with the Google/Samsung Galaxy Nexus i9250 for its compact size (though much larger than the iPhone 4s) and a decent deal I found on ebay. It was released in 2011 and has 16 gb of storage and generally low specs from a modern perspective. For me this is no problem as I'll mostly be using it as a dumb phone, glorified pager, ssh client, and maybe an audio player.

Installation

INSTRUCTIONS ARE SPECIFICALLY FOR THE GALAXY NEXUS i9250, THEY WILL NOT WORK FOR OTHER DEVICESs

For this guide I am assuming the your are using and reasonably familiar with GNU/Linux.

0. How it works

1. Download and Very Recovery and OS Images

In order to ensure that the files you download were not corupted or tampered with, add the release key to you keychain, as per this guide. I'm using Replicant 6.0, so I want the key with ID 5816A24C10757FC4. 

# Import the key
gpg --recv-keys 5816A24C10757FC4

Next, download the apropriate recovery image, the OS image, their checksum file, and signature files from here. For me they are:

  • replicant-6.0-maguro.zip – The OS.
  • replicant-6.0-maguro.zip.asc – The OS signature file.
  • recovery-maguro.img – The recovery.
  • recovery-maguro.img.asc – The recovery signature file.
  • maguro.sha256 – The checksum file for the OS and recovery.

Verify the recovery and OS signature files. If you do not see good signature DO NOT PROCEDE, the files have been tampered with. 

# Recovery
gpg --verify recovery-maguro.img.asc
gpg --verify replicant-6.0-maguro.zip.asc

You next should verify the checksums of the OS and recovery files to ensure that they have not been corrupted in transit. In order to avoid tedious manual comparison between the output of sha256sum and the content of maguro.sha256, I recommend using the script below.

If all went well, you're all set to begin flashing the recovery!

2. Flash Recovery Image
